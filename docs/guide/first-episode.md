# 做完第一期节目

假设你已经按 [快速开始](./getting-started.md) 启动并完成账号初始化。

## 1. 确认 AI 服务

打开 **设置 → AI 服务**（分区说明见 [设置中心](./settings.md)）（或检查 `.env` 中的 `OPENAI_*`）：

- API Key / Base URL 可用  
- Chat / ASR / TTS 模型 ID 与提供方一致  
- 需要封面时再配 Image 模型（可选）

也可用 MCP 工具 `get_system_health` 查看当前配置是否就绪。

## 2. （可选）调人设与音色

| 设置 | 作用 |
| --- | --- |
| **人设** | 主播是谁、对谁讲、节目气质 |
| **音色** | 默认 TTS 声音；可预置或 Voice Design |
| **提示词** | 口播 / 封面 / 闪卡模板，进阶再改 |

不改也能跑：系统有合理默认。

## 3. 丢进一条内容

首页 / 创建入口支持：

- **链接**：公开文章、可直链的音视频  
- **本地文件**：视频 / 音频 / 文稿  
- **纯文本**：会议纪要、笔记粘贴  

可选：指定 Source 插件、专辑、本集人设 / 音色 / 语言。

## 4. 等流水线跑完

典型阶段：

```text
获取素材 → 提取音频 → ASR 转写 → 口播稿
       → 封面 / 闪卡 → TTS 合成 → 可上架听播库
```

- 首页可看进度与状态  
- 失败可 **重试** 或从指定步骤重跑（跳过已完成）  
- 文稿类输入会跳过 ASR，直接进入写稿  

细节见 [制作流水线](./pipeline.md)。

## 5. 听与整理

- 任务详情：转写、口播稿、笔记、闪卡、封面、音频  
- **听播库**：上架后的沉浸播放（进度、倍速、睡眠定时）  
- **专辑 / 星图**：按主题收纳与回看  

## 6. 进阶玩法

- [定时订阅](./schedule.md)：RSS / 榜单自动进匣  
- [MCP 接入](./mcp.md)：用 Cursor 一句话创建节目  
- [插件体系](../plugins/)：扩展内容源与音色  

## 常见卡点

| 现象 | 排查 |
| --- | --- |
| 一直失败在转写 | ASR 模型 / Key；音视频是否可解 |
| 稿子很干或很短 | 原文太短；调人设与口播提示词 |
| 没有声音 | TTS 模型与音色名；看任务错误信息 |
| 链接抓不到 | 需登录墙 / 非直链 → 试 Source 插件或改本地上传 |

更多见 [FAQ](./faq.md)。
